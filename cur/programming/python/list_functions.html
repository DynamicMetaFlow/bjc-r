Append

There are many list functions that mutate rather than recreate a list. The first of these is the list append function. This operation adds an element to the end of the list (Note:this is different from append block in Snap!, which will combine lists). This function cannot be called alone. It must be called using a existing list as shown below. This is an example of the object oriented programming paradigm: you can think of the append function as a superpower that the list uses to absorb items. This is quite different from Snap!, where the add block is a contractor who can add items to any list upon request.

As an analogy, in Python, lists can feed themselves. You put "pod" in the list's mouth and tell it to eat, and it does. By contrast, in Snap!, you have to put the pod right into the poor list's tummy yourself.


>>> idiom = ["peas", "in", "a"]
>>> idiom.append("pod")
                    

>>> idiom
["peas", "in", "a", "pod"]

Note: idiom.append("pod") does not return the modified list. In other words, it's like a Snap! command, not a Snap! reporter. If you've just been reading the lab so far, try this example out and make sure you see what we mean by this note.

Insert

If you want the list to add the item to a specific location instead of the end, you'd use the insert function to place an element at a desired index. The first argument is the index of the insert position, the second argument is the element itself. 


>>> remark = ["break", "leg"]
>>> remark.insert(1, "a")
                    

>>> remark
["break", "a", "leg"]

Pop

Finally, use pop(i) to remove the item at position i in the list. If no argument is given (e.g. crime.pop()) will remove the last item of the list. 


>>> crime = ["dont", "steal", "the", "cookie"]
>>> crime.pop()
'cookie'
                    

>>> crime
["dont", "steal", "the"]

Note: in Snap! the delete block does not return the list item that is deleted, while Python's .pop() function does return the item that it removes. If you'd like, you could make a Snap! reporter that takes in a list, an mimics Python's pop() function.
