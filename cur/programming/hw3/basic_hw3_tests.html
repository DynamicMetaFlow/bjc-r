How The Test Block Works


We've provided a large number of blocks that you can use in HW3. You'll also need to fill in all of the gray blocks described in the spec. As mentioned in lecture, in this assignment we're going to use a new HoF called test to verify the correct function of the gray blocks that we build.

To understand how the testing block works, let's start by looking at some tests for blocks that are already completed.

Click on the "TESTER" sprite, and you'll see two example tests for the addition operation. The first block tests that 1 + 1 = 2. Try running it, and you shuold get back "true", since this test is correct. The second block performs two tests: One that 1 + 1 = 2, and one that 2 + 4 = 5. Try running it, and you should get the first test passes and the second test fails.

In this specific case, the problem is not the addition operation, but the input/output relationships that were wrong. In other words, we wrote an intentionally buggy test so you could see what happens when there is an input/output mismatch.
